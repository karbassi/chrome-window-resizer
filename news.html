History

Version 1.7.0 (2011-05-14):

  • Added the resize options as a context menu (right-click) for pop-up windows, when the toolbar icon is not available.
  • Added option to center window on screen when resized.
    This can be set for each preset individually, or you can set it globally from the "Settings" page (overriding everything else).

Version 1.6.2 (2011-03-31):

  • Minor bugs fixed.

Version 1.6.1 (2011-03-30):

  • Improved mechanism for resizing the viewport. Previously, if the window was maximized when trying to resize the viewport, there was a bug that caused the extension to set negative values for the width and height, and the window ended being 0 pixels wide.
  • The extension now needs the permission to access the content on all pages. This is required in order for the extension's option to resize just the viewport to work in the latest Chrome dev releases.

    Technical info:
    In order to resize the viewport, the extension needs to know the size of the window's borders, but, unfortunetly, this isn't available through the API. So, in order to determine the borders' size, the extension reads the width and height of the actual window, then it takes a screenshot of the visible area of the current tab and it substracts the size of the generated image from the window's size.
    Unfortunetly, the latest dev versions of Chrome require the extension to ask for permission to access the content of any given tab in order for it to be able to take a screenshot of that tab using the API and that's why it stopped working.

Version 1.6.0 (2010-10-05):

  • Added the option to use the current window's position.
    Check "Use current window position" and move the window to the desired location on the screen to update the Left/Top values.
    Many thanks to Paul H. for the suggestions! ;)
  • Added a way to export your settings so you can import them in another (Chrome) browser. This way, you can easily copy your settings from your home computer to your office or whatever... :)
  • Fixed bug that was preventing to update the window's position for resolutions defined as "mobile".

Version 1.5.0 (2010-10-04):

  • Added new option on the settings page to control the behavior of the extension's icon when clicked.
    You can choose to display the default menu (the resolutions list) or to cycle through the defined resolutions.
  • Added the option to set the window's position by specifying the X,Y coordinates (the left and top position).

Version 1.4.4 (2010-09-14):

  • Fixed some bug with adding/editing resolutions.
    When adding resolutions, the script failed to assign a unique ID and if you tried to edit any of those the script would assign the new values to all the new resolutions in the list.
    However, the settings were saved correctly and if you closed the settings page and opened it again, the resolutions list would display correctly.
    Thanks to Nate Jr. (n8) for taking the time to report this bug!

Version 1.4.3 (2010-08-22):

  • Fixed issue with window size overlay appearing on printed pages.
    Thanks to user juliog for reporting it and also providing a solution to the problem!
  • The extension will try to detect if the current page is a view-source page or a media file (image, mp3, video, etc.) before trying to insert the window size overlay in the bottom-right corner.
  • NOTE FOR WEBSITES DEVELOPERS
    Users Vinz and Ryan also reported an issue related not only to this extension, but to all extensions (E.g.: MeasureIt) that are using extension specific content scripts (scripts that are altering the DOM or adding CSS/JS to the page to perform certain functions; E.g.: overlays).
    As far as I can tell, this issue appears on websites using the head:first-child + body CSS selector to apply certain styles only on some browsers.
    The problem is that Chrome (wrongly) inserts a <style> tag right before the <head> (instead of inside) to add those content scripts, making the aforementioned CSS selector useless (and breaking your page if you're using it) since <head> is not the first-child anymore...

Version 1.4.2 (2010-05-31):

  • Fixed duplicate page request when tooltip was enabled.
    Thanks to all those who reported the bug!

Version 1.4.1 (2010-04-20):

  • Fixed TypeError: Cannot call method 'remove' of undefined error displayed in the JavaScript console when tooltip was hidden.
    Thank you Dima for pointing it out to me!

Version 1.4.0 (2010-04-18):

  • The width and height are now displayed in the popup.
  • The "resolution title" has been renamed to "description" and has become optional.
  • Added a tooltip that's displayed in the bottom right corner of the page when resizing the window. (only available on pages loaded via the HTTP or HTTPS protocols; it won't work on local files, extensions settings pages, the new tab page, the Chrome extensions gallery or any other special page)
  • Added new settings page, where you can control several options, like whether to display or not the tooltip when resizing, the duration the tooltip is displayed, whether to display or not the resolution description in the popup and the width of the popup.
  • Minor tweaks added to the interface.

Version 1.3.0 - 1.3.1 (2010-04-13):

  • Added the option to choose between custom, current window or current viewport size when adding a new resolution preset.
  • Added the "Updated!" button in the resolutions popup, which will appear after any new updates (and disappear after clicking on it).
  • Added this history/news page.

  • Fixed minor bugs
